首先，回顾一下内存分配。所有程序都必须预留足够的内存来存储程序使用的数据。这些内存中有些是自动分配的。例如：
```
float x;
char place[] = "sichuan";
```
为一个float类型的值和一个字符串预留了足够的内存，或者可以显式指定分配一定数量的内存：
```
int plates[100];
```
该声明预留了100个内存位置，每个位置都用于存储int类型的值。声明还为内存提供了一个标识符，因此可以用x和place识别数据。  
静态数据在程序载入内存时分配，而自动数据在程序执行块时分配，并在程序离开该块时销毁。

C能做的不止这些。可以在程序运行时分配更多的内存。主要的工具是`malloc()`函数，该函数接受一个参数：所需的内存字节数。  
`malloc()`函数会找到合适的空闲内存块，这样的内存都是匿名的。也就是说，`malloc()`分配内存，但是不会为其赋名。  
然而，它确实返回动态分配内存块的首字节地址。因此，可以把该地址赋给一个指针变量，使用指针访问这块内存。

为啥要使用malloc来动态申请内存，觉得关键有两点：
1.很多情况不能确定需要多大的空间来存数据结构，虽然有变长数组，但是变长数组只是一个大小可以用变量表示的结构，编译期不确定大小，但是执行期一旦执行了，仍然是不能改变长度的，而且不能主动释放。所以需要malloc动态申请内存和释放内存。
2.全局变量或者局部变量的生命周期都是程序员无法控制的，例如局部变量在函数出栈时生命周期就结束。但是例如链表结构，添加节点和删除节点（申请内存和释放内存）都需要程序员主动来控制，所以必须用malloc和free。
